PANTA & MJ 项目:

关于清除历史数据，需要在现有的基础上添加下面的清除Task:

1）在清除JHF_SPOT_RATE_HISTORY时同时清除JHF_DL_MANUAL_RATE_HISTORY;
   这一点添加到清除BATCH中。


2) 下面表的历史数据除了查BUG以外，没有什么用处。需要运营来编写脚本，
   在每周的EOW将2个WEEK前的数据备份成文件和到本地DataBase中，然后DELETE掉。

   SELECT * FROM JHF_ORDER_STATUS_HISTOTRY WHERE UPDATE_DATE IN ( 2周前） 
   SELECT * FROM JHF_CONTRACT_STATUS_HISTORY WHERE UPDATE_DATE IN ( 2周前） 
   SELECT * FROM JHF_DEPOSIT_STATUS_HISTORY  WHERE UPDATE_DATE IN ( 2周前）
   SELECT * FROM JHF_WITHDRAWAL_STATUS_HISTORY  WHERE UPDATE_DATE IN ( 2周前）

   SELECT * FROM JHF_CUSTOMER_MAIL_HISTORY WHERE UPDATE_DATE IN ( 2周前） 
   SELECT * FROM JHF_SYS_MAIL_HISTORY WHERE UPDATE_DATE IN ( 2周前） 
   SELECT * FROM JHF_HEDGE_CUSTTRADE_HISTORY WHERE UPDATE_DATE IN  ( 2周前） 

   SELECT * FROM JHF_MARGIN_ACCOUNT_DAY_HISTORY WHERE FRONT_DATE IN ( 2周前） 
   SELECT * FROM JHF_TASK_HISTORY WHERE FRONT_DATE IN ( 2周前） 
  

3) 下面这个表每周备份出来后TUNCATE即可。
   JHF_CLEAR_CACHE_MANAGE



FES 项目:

1）在清除JHF_SPOT_RATE_HISTORY时同时清除JHF_DL_MANUAL_RATE_HISTORY;
   这一点添加到清除BATCH中。


2) 下面表的历史数据除了查BUG以外，没有什么用处。需要运营来编写脚本，
   在每个季度的第一周EOW,将一个季度之前的履历数据备份到后备数据库中，并进行清除：
   SELECT * FROM JHF_ORDER_STATUS_HISTOTRY WHERE UPDATE_DATE IN ( 2周前） 
   SELECT * FROM JHF_CONTRACT_STATUS_HISTORY WHERE UPDATE_DATE IN ( 2周前） 
   SELECT * FROM JHF_DEPOSIT_STATUS_HISTORY  WHERE UPDATE_DATE IN ( 2周前）
   SELECT * FROM JHF_WITHDRAWAL_STATUS_HISTORY  WHERE UPDATE_DATE IN ( 2周前）

   SELECT * FROM JHF_CUSTOMER_MAIL_HISTORY WHERE UPDATE_DATE IN ( 2周前） 
   SELECT * FROM JHF_SYS_MAIL_HISTORY WHERE UPDATE_DATE IN ( 2周前） 
   SELECT * FROM JHF_HEDGE_CUSTTRADE_HISTORY WHERE UPDATE_DATE IN  ( 2周前） 

   SELECT * FROM JHF_MARGIN_ACCOUNT_DAY_HISTORY WHERE FRONT_DATE IN ( 2周前） 
   SELECT * FROM JHF_TASK_HISTORY WHERE FRONT_DATE IN ( 2周前） 

以上(2)(3)内容，如果没有备份数据库，也可以在备份出来后，直接DELETE.

3) 下面这个表每周备份出来后TUNCATE即可。
   JHF_CLEAR_CACHE_MANAGE
